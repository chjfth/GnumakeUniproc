[2009-05-05] Chj Note:
I sugguest using umake to run the Makefile, not using umakeD.
With umakeD.bat(today), gmp_bc_DEBUG=1 is set on CMD.EXE command line as make's parameter, so it will override ``gmp_bc_DEBUG='' settings with <refname>_MakeVarDefines assignment.
Example: In $gmu_DIR_ROOT\GMU-examples\windows\CopyPathExt\make-all-variant\u_subprjs.mki , there are gmp_bc_DEBUG=(null) assignments, but they will result in vain with umakeD.bat, and the result is: two sub-projects will result in getting the same project-name. Since such two-project-same-name problem is from the same parent project, it will not cause make to halt(today's behavior), but outputs warning such as:
