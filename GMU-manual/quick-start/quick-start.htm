GnumakeUniproc Quick Start Guide

This quick start guide intends to tell you how to setup the environment to work with GnumakeUniproc. Our goal is to build our first sample program(including a LIB project and an EXE project) with GnumakeUniproc.

This guide is based on GnumakeUniproc v0.96. GnumakeUniproc is often abbreviated as GMU.

GnumakeUniproc can be run on Windows or Linux, in detail:

  • On Windows 2000, Windows XP, Windows Vista SP1 or SP2(but broken on Vista without SP), Windows 7.
  • On Linux, or probably on Unix variants with Bash shell installed(sh should link to Bash).

Installation

Basic software environment prerequisites:

  1. You always need GNU make version 3.80 or above. Currently(May 2010) latest GNU make version is 3.81, which is strongly recommended over 3.80 to avoid subtle problems.
    For GMU's Windows installer, make 3.81 is already bundled for you.
  2. If you use GMU's SVN repository auto-checkout feature, you need at least Subversion 1.4.2 .

More about the example program

The example program in the quick start guide only exhibits a small portion of GnumakeUniproc's capability. To fully exploit GnumakeUniproc's functionality, refer to its manual please(give a link later).

If you have decided to deploy GnumakeUniproc for your own C/C++ projects, you may start writing your own makefiles based on some makefile templates. Those templates reside in:

$gmu_DIR_ROOT/GMU-manual/gmu-template
Throughout GnumakeUniproc's documents , $gmu_DIR_ROOT is used to represent where GMU is installed. Actually gmu_DIR_ROOT is a real environment variable(env-var for short) you can refer to after GMU installation.

The walkdir LIB project is based on template in: 

$gmu_DIR_ROOT/GMU-manual/gmu-template/use-pattern1/make-lib

and the walkdir EXE project is based on template in: 

$gmu_DIR_ROOT/GMU-manual/gmu-template/use-pattern1/make-exe

More information can be found on GnumakeUniproc's website .
